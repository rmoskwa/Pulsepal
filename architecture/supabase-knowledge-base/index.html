    PulsePal Supabase Knowledge Base Architecture¶

    Overview¶

    The PulsePal project utilizes a Supabase PostgreSQL database as its knowledge base for storing and retrieving MRI sequence programming information. The database leverages vector embeddings for semantic search capabilities and maintains a comprehensive collection of Pulseq documentation, API references, code examples, and complete sequence implementations.

    Project ID: mnbvsrsivuuuwbtkmumt
    Last Updated: January 2025

    Database Schema¶

    Core Tables¶

    The knowledge base consists of six primary tables that work together to provide comprehensive MRI sequence programming support:

    1. sources Table¶

    Tracks all data sources that have been indexed into the knowledge base.

    Column Type Description
    source_id text (PK) Unique identifier for the source (e.g., GitHub repo URL)
    summary text Brief description of the source content
    total_word_count integer Total word count of all content from this source
    created_at timestamptz When the source was first indexed
    updated_at timestamptz Last update timestamp

    Statistics: - Total sources: 27 - Storage size: 16 KB - Primary content: GitHub repositories containing Pulseq implementations

    2. crawled_pages Table¶

    Main content storage for documentation and code examples.

    Column Type Description
    id bigint (PK) Unique identifier
    url varchar Source URL of the content
    chunk_number integer Chunk sequence number for paginated content
    content text Actual text content
    source_id text Foreign key to sources table
    metadata jsonb Additional metadata (tags, categories, etc.)
    created_at timestamptz Indexing timestamp
    embedding vector(768) 768-dimensional vector embedding for semantic search
    sequence_category varchar Classification of sequence type (e.g., 'GRE', 'EPI', 'MPRAGE')

    Statistics: - Total rows: 1,427 - Storage size: 2.6 MB - Contains chunked documentation and code examples - Includes official Pulseq sequence implementations

    3. api_reference Table¶

    Comprehensive API function reference for Pulseq/pypulseq with enhanced metadata.

    Column Type Description
    id bigint (PK) Unique identifier
    name text Function/method name
    language varchar Programming language ('matlab' or 'python')
    signature text Function signature
    description text Detailed description
    parameters jsonb Parameter details (names, types, descriptions)
    returns jsonb Return value information
    source_id text Source reference
    pulseq_version varchar Pulseq version compatibility
    embedding vector(768) Vector embedding for semantic search
    function_type varchar Type classification ('main', 'helper', etc.)
    usage_examples jsonb Code examples demonstrating usage
    related_functions jsonb List of related API functions
    has_nargin_pattern boolean MATLAB nargin pattern detection
    class_name text Parent class name (if method)
    is_class_method boolean Whether function is a class method
    calling_pattern text Common calling patterns
    instance_variable text Common instance variable name
    search_terms text[] Array of search keywords
    class_metadata jsonb Additional class information
    last_updated timestamp Last update timestamp

    Statistics: - Total rows: 128 - Storage size: 544 KB - Covers both MATLAB and Python implementations - Includes complete function signatures and usage patterns

    4. pulseq_sequences Table (NEW)¶

    Complete Pulseq sequences with rich metadata for intelligent retrieval.

    Column Type Description
    id bigserial (PK) Unique identifier
    file_name text Name of the sequence file
    full_path text Full file path in repository
    repository text Source repository URL
    full_code text Complete sequence code
    line_count integer Number of lines in sequence
    language text Programming language (default: 'matlab')
    sequence_family text Primary type (GRE, EPI, TSE, PRESS, etc.)
    contrast_mechanism text Contrast generation method
    trajectory_type text K-space sampling pattern
    dimensionality text 0D, 1D, 2D, 3D classification
    architecture_type text Code organization (single-file/modular/workflow)
    complexity_level integer Complexity rating (1-5)
    dependencies jsonb References to helper files
    external_requirements text[] External dependencies
    preparation_techniques text[] Preparation methods used
    acceleration_methods text[] Acceleration techniques
    advanced_features text[] Advanced sequence features
    educational_value text Educational rating (high/medium/low)
    typical_applications text[] Clinical applications
    hardware_requirements text[] Scanner requirements
    embedding vector(768) Semantic search embedding
    created_at timestamptz Creation timestamp

    Statistics: - Total rows: 5 - Storage size: 8 KB - Complete sequence implementations with analysis

    5. sequence_chunks Table (NEW)¶

    Code sections from sequences following universal pattern recognition.

    Column Type Description
    id bigserial (PK) Unique identifier
    sequence_id bigint Reference to pulseq_sequences
    chunk_type text Section type (parameter_definition, rf_gradient_creation, etc.)
    chunk_order integer Order within sequence
    start_line integer Starting line number
    end_line integer Ending line number
    percentage_of_sequence integer Percentage of total sequence
    is_pulseq boolean True for sequence code, false for reconstruction
    code_content text Actual code for this section
    pulseq_functions text[] Pulseq API functions used
    key_concepts text[] MRI concepts in section
    key_variables text[] Important variables
    mri_concept text Primary MRI concept
    description text Human-readable description
    embedding vector(768) Section-specific embedding
    created_at timestamptz Creation timestamp

    Statistics: - Total rows: 34 - Storage size: 40 KB - Granular code sections for targeted retrieval

    6. crawled_code Table (NEW)¶

    All sequence-related code files including helpers and dependencies.

    Column Type Description
    id bigserial (PK) Unique identifier
    content text Full file content
    source_id text Repository/source identifier
    content_type text File classification (main_sequence, helper_function, etc.)
    parent_sequences bigint[] Array of sequences using this file
    dependency_type text Required or optional dependency
    file_name text Name of the file
    file_extension text File extension
    language_detected text Detected programming language
    line_count integer Number of lines
    pulseq_functions_used text[] Pulseq functions in file
    custom_functions_defined text[] Functions defined in file
    imports_files text[] Files imported/required
    metadata jsonb Content-specific metadata
    embedding vector(768) File-level embedding
    created_at timestamptz Creation timestamp

    Statistics: - Total rows: 15 - Storage size: 24 KB - Supporting files and dependencies

    Database Views¶

    1. official_sequence_examples¶

    Filtered view of crawled_pages containing official Pulseq sequence examples with embeddings.

    2. api_reference_search¶

    Optimized view for searching API functions with enhanced metadata.

    3. api_reference_details¶

    Detailed view combining API reference with additional context.

    4. function_calling_patterns¶

    Analysis view showing common function usage patterns extracted from code examples.

    Vector Search Infrastructure¶

    Extensions Enabled¶

    1. pgvector (v0.8.0) - Core vector similarity search
    2. Supports 768-dimensional embeddings
    3. Enables cosine similarity, L2 distance, and inner product operations
    4. Used for semantic search across all content

    5. Optimized indexes on embedding columns

    6. pg_stat_statements - Query performance monitoring

    7. uuid-ossp - UUID generation for unique identifiers
    8. pgcrypto - Cryptographic functions for secure operations

    RPC Functions for Search¶

    Core Search Functions¶

    1. match_crawled_pages
    2. Semantic search across documentation and code examples
    3. Parameters: query_embedding (vector), match_threshold (float), match_count (int)
    4. Returns: Similar content with similarity scores

    5. Searches the crawled_pages table

    6. match_api_reference

    7. Search API functions by semantic similarity
    8. Parameters: query_embedding (vector), match_threshold (float), match_count (int)
    9. Includes function signatures, descriptions, and examples

    10. Optimized for finding relevant Pulseq functions

    11. match_api_reference_search

    12. Enhanced API search with additional filtering
    13. Parameters: query_embedding (vector), match_threshold (float), match_count (int)
    14. Supports language-specific searches (MATLAB/Python)

    15. Uses the api_reference_search view

    16. match_official_sequences

    17. Search official Pulseq sequence implementations
    18. Parameters: query_embedding (vector), match_threshold (float), match_count (int)
    19. Filtered by sequence category

    20. Returns complete sequence examples

    21. match_code_examples

    22. Find relevant code snippets and examples
    23. Parameters: query_embedding (vector), match_threshold (float), match_count (int)
    24. Links to source repositories
    25. Searches crawled_pages with code content

    Access Patterns¶

    1. Hybrid Search Strategy¶

    The system implements a hybrid search combining: - Vector similarity search (semantic understanding) - Keyword matching (exact term matching) - Category filtering (sequence types, languages)

    2. Intelligent Routing¶

    Based on query analysis: - Documentation queries crawled_pages - Function lookups api_reference - Sequence examples official_sequence_examples - Code patterns function_calling_patterns

    3. Performance Optimization¶

    • Indexed vector columns for fast similarity search
    • Materialized views for complex queries
    • Cached embeddings to avoid recomputation

    Security Configuration¶

    • Row Level Security (RLS): Enabled on all tables
    • API Key Authentication: Via Supabase service key
    • SSL/TLS: Enforced for all connections
    • Access Control: Read-only access for client applications

    Integration with PulsePal¶

    RAG Service Integration (rag_service.py)¶

    The RAG service interfaces with this knowledge base through:

    1. Supabase Python Client: Direct API access
    2. Google Embeddings API: Generates query embeddings
    3. RPC Function Calls: Executes similarity searches
    4. Result Processing: Formats and ranks results

    Enhanced Architecture Features (NEW)¶

    Three-Tier Table Structure¶

    The enhanced database architecture implements a three-tier structure for optimal retrieval:

    1. Complete Sequences (pulseq_sequences)
    2. Full implementations with rich metadata
    3. Classification across multiple dimensions

    4. Feature arrays for smart filtering

    5. Granular Sections (sequence_chunks)

    6. Universal 6-section pattern recognition
    7. Function and concept extraction

    8. Section-specific embeddings

    9. Supporting Files (crawled_code)

    10. Helper functions and utilities
    11. Dependency tracking
    12. Cross-reference maintenance

    Advanced Search Capabilities¶

    • Multi-dimensional Classification: Sequences classified by family, contrast, trajectory, and dimensionality
    • Feature-based Filtering: Arrays for preparation techniques, acceleration methods, and advanced features
    • Dependency Resolution: Automatic tracking of file dependencies and relationships
    • Educational Value Assessment: Sequences rated for teaching effectiveness

    Common Queries¶

    -- Find all GRE sequences in crawled_pages
SELECT * FROM crawled_pages 
WHERE sequence_category = 'GRE';

-- Get complete sequences by type
SELECT * FROM pulseq_sequences 
WHERE sequence_family = 'EPI';

-- Get function by name
SELECT * FROM api_reference 
WHERE name ILIKE '%addBlock%' 
AND language = 'matlab';

-- Find sequence chunks by type
SELECT * FROM sequence_chunks 
WHERE chunk_type = 'rf_gradient_creation';

-- Get helper functions for a sequence
SELECT cc.* FROM crawled_code cc
WHERE cc.content_type = 'helper_function'
AND cc.parent_sequences @> ARRAY[1::bigint];

-- Search with embeddings (via RPC)
SELECT * FROM match_crawled_pages(
    query_embedding := '[...]'::vector,
    match_threshold := 0.7,
    match_count := 5
);

    PulsePal Supabase Knowledge Base Architecture¶

    Overview¶

    The PulsePal project utilizes a Supabase PostgreSQL database as its knowledge base for storing and retrieving MRI sequence programming information. The database leverages vector embeddings for semantic search capabilities and maintains a comprehensive collection of Pulseq documentation, API references, and code examples.

    Project ID: mnbvsrsivuuuwbtkmumt

    Database Schema¶

    Core Tables¶

    1. sources Table¶

    Tracks all data sources that have been indexed into the knowledge base.

    Column Type Description
    source_id text (PK) Unique identifier for the source (e.g., GitHub repo URL)
    summary text Brief description of the source content
    total_word_count integer Total word count of all content from this source
    created_at timestamptz When the source was first indexed
    updated_at timestamptz Last update timestamp

    Current Sources: 27 sources indexed, primarily GitHub repositories containing Pulseq implementations and examples.

    2. crawled_pages Table¶

    Main content storage for documentation and code examples.

    Column Type Description
    id bigint (PK) Unique identifier
    url varchar Source URL of the content
    chunk_number integer Chunk sequence number for paginated content
    content text Actual text content
    source_id text Foreign key to sources table
    metadata jsonb Additional metadata (tags, categories, etc.)
    created_at timestamptz Indexing timestamp
    embedding vector 768-dimensional vector embedding for semantic search
    sequence_category varchar Classification of sequence type (e.g., 'GRE', 'EPI', 'MPRAGE')

    Statistics: - Total rows: 1,703 - Storage size: ~54 MB - Contains chunked documentation and code examples

    3. api_reference Table¶

    Comprehensive API function reference for Pulseq/pypulseq.

    Column Type Description
    id bigint (PK) Unique identifier
    name text Function/method name
    language varchar Programming language ('matlab' or 'python')
    signature text Function signature
    description text Detailed description
    parameters jsonb Parameter details (names, types, descriptions)
    returns jsonb Return value information
    source_id text Source reference
    pulseq_version varchar Pulseq version compatibility
    embedding vector Vector embedding for semantic search
    function_type varchar Type classification ('main', 'helper', etc.)
    usage_examples jsonb Code examples demonstrating usage
    related_functions jsonb List of related API functions
    has_nargin_pattern boolean MATLAB nargin pattern detection
    class_name text Parent class name (if method)
    is_class_method boolean Whether function is a class method
    calling_pattern text Common calling patterns
    instance_variable text Common instance variable name
    search_terms text[] Array of search keywords
    class_metadata jsonb Additional class information
    last_updated timestamp Last update timestamp

    Statistics: - Total rows: 413 - Storage size: ~8.8 MB - Covers both MATLAB and Python implementations

    Database Views¶

    1. official_sequence_examples¶

    Filtered view of crawled_pages containing official Pulseq sequence examples with embeddings.

    2. api_reference_search¶

    Optimized view for searching API functions with enhanced metadata.

    3. api_reference_details¶

    Detailed view combining API reference with additional context.

    4. function_calling_patterns¶

    Analysis view showing common function usage patterns extracted from code examples.

    Vector Search Infrastructure¶

    Extensions Enabled¶

    1. pgvector (v0.8.0) - Core vector similarity search
    2. Supports 768-dimensional embeddings
    3. Enables cosine similarity, L2 distance, and inner product operations

    4. Used for semantic search across all content

    5. pg_stat_statements - Query performance monitoring

    6. uuid-ossp - UUID generation for unique identifiers
    7. pgcrypto - Cryptographic functions for secure operations

    RPC Functions for Search¶

    Core Search Functions¶

    1. match_crawled_pages
    2. Semantic search across documentation and code examples
    3. Parameters: query_embedding, match_threshold, match_count

    4. Returns: Similar content with similarity scores

    5. match_api_reference

    6. Search API functions by semantic similarity
    7. Includes function signatures, descriptions, and examples

    8. Optimized for finding relevant Pulseq functions

    9. match_api_reference_search

    10. Enhanced API search with additional filtering

    11. Supports language-specific searches (MATLAB/Python)

    12. match_official_sequences

    13. Search official Pulseq sequence implementations
    14. Filtered by sequence category

    15. Returns complete sequence examples

    16. match_code_examples

    17. Find relevant code snippets and examples

    18. Links to source repositories

    19. search_functions_by_capability

    20. Capability-based function discovery
    21. Maps user intent to specific API functions

    Access Patterns¶

    1. Hybrid Search Strategy¶

    The system implements a hybrid search combining: - Vector similarity search (semantic understanding) - Keyword matching (exact term matching) - Category filtering (sequence types, languages)

    2. Intelligent Routing¶

    Based on query analysis: - Documentation queries crawled_pages - Function lookups api_reference - Sequence examples official_sequence_examples - Code patterns function_calling_patterns

    3. Performance Optimization¶

    • Indexed vector columns for fast similarity search
    • Materialized views for complex queries
    • Cached embeddings to avoid recomputation

    Security Configuration¶

    • Row Level Security (RLS): Enabled on all tables
    • API Key Authentication: Via Supabase service key
    • SSL/TLS: Enforced for all connections
    • Access Control: Read-only access for client applications

    Integration with PulsePal¶

    RAG Service Integration (rag_service.py)¶

    The RAG service interfaces with this knowledge base through:

    1. Supabase Python Client: Direct API access
    2. Google Embeddings API: Generates query embeddings
    3. RPC Function Calls: Executes similarity searches
    4. Result Processing: Formats and ranks results

    Key Integration Points¶

    # Example: Searching for API functions
results = await supabase.rpc(
    'match_api_reference',
    {
        'query_embedding': embedding,
        'match_threshold': 0.7,
        'match_count': 10
    }
).execute()

    Common Queries¶

    -- Find all GRE sequences
SELECT * FROM official_sequence_examples 
WHERE sequence_category = 'GRE';

-- Get function by name
SELECT * FROM api_reference 
WHERE name ILIKE '%addBlock%' 
AND language = 'matlab';

-- Search with embeddings (via RPC)
SELECT * FROM match_crawled_pages(
    query_embedding := '[...]'::vector,
    match_threshold := 0.7,
    match_count := 5
);

    Always test vector search queries with representative examples to ensure: - Appropriate similarity thresholds - Reasonable result counts - Performance within acceptable limits
